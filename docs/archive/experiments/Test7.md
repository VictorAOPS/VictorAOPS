<details>
  <summary><b>Cycle Time — DHRS closure (30 → 5 min)</b></summary>
  <br/>
  <ul>
    <li><b>Context:</b> cierres con fricción (pasos manuales + baja visibilidad).</li>
    <li><b>Action:</b> estandarización del flujo + señalización de pendientes + tablero para seguimiento.</li>
    <li><b>Result:</b> reducción de ciclo y menor variación del cierre.</li>
    <li><b>Evidence:</b> timestamps del flujo + KPI de ciclo + bitácora de seguimiento.</li>
  </ul>
</details>

<details>
  <summary><b>Admin Load — PM workflow (↓ 40% time)</b></summary>
  <br/>
  <ul>
    <li><b>Context:</b> carga administrativa + trazabilidad dispersa.</li>
    <li><b>Action:</b> digitalización (checklist + evidencia + estados) y estándar de captura.</li>
    <li><b>Result:</b> menos tiempo administrativo y registro más consistente.</li>
    <li><b>Evidence:</b> tiempos de registro antes/después + completitud + trazabilidad por OT.</li>
  </ul>
</details>

<details>
  <summary><b>Discrepancies — Tool Crib 5S (↓ 25%+ variance)</b></summary>
  <br/>
  <ul>
    <li><b>Context:</b> variación inventario físico vs. registro → retrabajo.</li>
    <li><b>Action:</b> 5S + gestión visual + reglas de control (quién/qué/cuándo).</li>
    <li><b>Result:</b> menos discrepancias y mayor predictibilidad.</li>
    <li><b>Evidence:</b> conteos cíclicos + hallazgos + tendencia de variación.</li>
  </ul>
</details>

<details>
  <summary><b>Planning Confidence — Supply chain (multi-source visibility)</b></summary>
  <br/>
  <ul>
    <li><b>Context:</b> información fragmentada (órdenes, inventarios, movimientos).</li>
    <li><b>Action:</b> consolidación multi-fuente + modelo de datos + dashboard con señales.</li>
    <li><b>Result:</b> coordinación mejorada y menor riesgo por falta de visibilidad.</li>
    <li><b>Evidence:</b> “single view” operacional + definiciones KPI + registro de decisiones.</li>
  </ul>
</details>
